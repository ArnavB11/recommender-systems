# Multi-Objective Recommendation System (MSRS vs. MSRS II)

This repository implements and evaluates multi-objective recommendation list optimization using PyTorch (for Neural Collaborative Filtering) and `pymoo` (for evolutionary search with NSGA-II). The system balances three conflicting stakeholder objectives:
1. **Personalized Genre Novelty (DOPM):** Balancing accuracy and diversity.
2. **BERT Semantic Serendipity:** Finding unexpected yet highly relevant recommendations using IMDb review text embeddings.
3. **Provider Exposure Fairness:** Distributing exposure across long-tail creators.

---

## MSRS vs. MSRS II: Core Differences

The two algorithms differ fundamentally in how they restrict the search space, seed their starting populations, and perform mutations.

| Feature / Phase | MSRS (Standard) | MSRS II (Exploration) |
| :--- | :--- | :--- |
| **Candidate Pre-filtering** | **PSNR Sweet-spot Filter:** Variant lists generated by weighted roulette are checked. Only lists whose NCF score variance falls in the 25th-75th percentile of the original list are kept. | **No Filter:** Disables the PSNR filter entirely, leaving the search pool wide open to explore the entire catalog. |
| **Initial Seeding ($P_0$)** | **Weighted Roulette Selection:** Starts with variants generated by sampling the top NCF items using rating predictions as probabilities. | **Novelty Seeding:** Seeds the initial population with two anchors: (1) the top accurate NCF list, and (2) the top popularity-novelty list. |
| **Mutation Operator** | **Standard Swap Mutation:** Randomly swaps two items within the recommendation list. | **Targeted Mutation (`MSRSIITargetedMutation`):** Replaces the item with the *lowest* predicted NCF score in the list with the *highest-novelty* (most obscure) item outside the list. |
| **Generations** | Runs for **40 generations** due to faster convergence. | Runs for **100 generations** to explore the unconstrained search space. |

## Stakeholder Performance Trade-off (at $K=10$)

* **The Consumer Stakeholder (Relevance):** 
  * **MSRS** is vastly superior. It maintains a **Precision@10 of 0.137** and a **Serendipity of 0.164** because the PSNR filter prevents low-rating items from cluttering the recommendations.
  * **MSRS II** drops to **0.073 Precision** and **0.139 Serendipity** because its unconstrained mutation introduces obscure movies that are too far from the user's taste.
* **The Provider Stakeholder (Exposure):**
  * **MSRS II** is the clear winner for niche content providers. It increases **Long-Tail Coverage to 11.51%** (a 13x increase over MSRS's 0.88%) and leverages **62.0% of the catalog** (vs. MSRS's 42.3%).
* **The System Stakeholder (Balance):**
  * **MSRS** offers a stable, user-safe platform balance (FAS Fairness: 0.917).
  * **MSRS II** pushes categorical balance (Genre Entropy: 2.33 bits vs. 2.05 bits in MSRS) but carries higher risk due to lower accuracy.
